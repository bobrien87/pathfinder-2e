# Import Workflow

This workflow details the procedures for checking and syncing creature data from the Foundry VTT PF2e repository to the Obsidian vault.

---

## Workflow Instructions

When the user triggers the `/import` command, execute these steps:

### Step 1: Determine the Mode
Check if the user provided arguments to the command (e.g. `/import --write` or `/import write`):
* **Dry Run Mode (Default):** If no arguments (or `--dry-run`) are provided, the workflow will inspect files and report updates without writing to disk.
* **Write Mode:** If `--write` is specified, the workflow will execute the full check and save new and updated creature notes to the vault.

### Step 2: Run the Import Script
Execute the import script using the Python command:

* **Dry Run Mode:**
  ```bash
  python3 ".agents/scripts/import_creatures.py"
  python3 ".agents/scripts/import_mechanics.py"
  ```

* **Write Mode:**
  ```bash
  python3 ".agents/scripts/import_creatures.py" --write
  python3 ".agents/scripts/import_mechanics.py" --write
  ```

*Note: The script automatically handles cloning or pulling the latest upstream Foundry VTT repository files inside the persistent directory `/Users/personal/.gemini/antigravity/pf2e`.*

### Step 3: Parse and Report Results
Read the output generated by the script and format it nicely for the user:

1. **Dry Run Output:**
   * Present the counts of New, Updated, Unchanged, and Ignored entities.
   * List the first 10 new and modified creatures so the user can review what has changed.
   * Inform the user that they can run `/import --write` to save these changes.

2. **Write Mode Output:**
   * Report the counts of successfully created new files and updated existing files.
   * Provide links to a sample of modified or created creature files.

### Step 4: Propose a Sync (Optional)
If any changes were written during Write Mode, suggest that the user run the `/sync` command to push the new and updated vault files to their remote GitHub repository.
